# MAGI Self-Verification Checklist

1. [ ] Codex/GPT-5とGeminiのCLIを事前疎通し、両者ともオンラインであることを確認した。
2. [ ] いずれかのCLIが失敗した場合、許可済みフォールバックを全て試し、それでも不可ならセッションを中断して報告した。
3. [ ] 3ペルソナの分析を並列で実行し、互いの内容を参照しない初期出力を取得した。
4. [ ] 各ペルソナの「Action – Tool/Owner – Success Check」箇条書きが3つずつ揃っている。
5. [ ] Gemini主導でUnified Action Plan（最低3ステップ、各ステップに検証方法付き）を作成した。
6. [ ] リスクは具体的なトリガーと対策のペアで記述し、フォローアップ質問は1件に絞った。
7. [ ] Situation Snapshotにゴール/制約/成功指標が明記されている。
8. [ ] 必要なら追加実装やテスト結果を該当ペルソナで提示し、ログやdiffを添えた。
9. [ ] `runtime/` 以下のログや補助ファイルを整理し、不要であれば削除した（詳細はSKILL.mdのクリーンアップ参照）。
10. [ ] 以上を満たした上で、MAGI SYSTEM STATUSを「DELiberation COMPLETE」または失敗理由に設定した。
